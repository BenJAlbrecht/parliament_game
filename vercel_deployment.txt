VERCEL DEPLOYMENT INSTRUCTIONS
===============================

No adapter changes needed — the project uses adapter-static with fallback: 'index.html',
which Vercel handles natively for client-side SvelteKit apps.


STEP 1 — Push current code to GitHub
--------------------------------------
Make sure main is up to date:

  git checkout main
  git merge dev
  git push origin main


STEP 2 — Create a Vercel account
----------------------------------
Go to https://vercel.com and sign up with your GitHub account.
This links the two accounts automatically.


STEP 3 — Import the repository
--------------------------------
1. From the Vercel dashboard, click Add New > Project
2. Find parliament_game in the repository list and click Import


STEP 4 — Configure the build
------------------------------
Vercel will auto-detect SvelteKit. Verify these settings:

  Framework Preset  : SvelteKit
  Build Command     : vite build
  Output Directory  : build
  Install Command   : npm install

Leave everything else as default.


STEP 5 — Deploy
----------------
Click Deploy. Vercel builds the project and gives you a live URL
(e.g. parliament-game.vercel.app) in about a minute.


ONGOING
--------
- Every push to main triggers an automatic redeploy.
- Pushes to other branches (e.g. dev) get a temporary preview URL,
  useful for testing before merging.
